Instructions for using a movie API and its possibilities

Request description endpoint URL Authorization y/n HTTP Method The format of data expected in the body of the request (and an example) The format of the response data
Create a new User in the list /users/ POST no need of Authorization JSON format of user to be added in the list { "Username": "username string", "Password": "password example", "Email": "domidomiddomi@mail.com", "Birthday": "1990/01/01" (date format), "FavMovies" : [] } JSON format with the new user created JSON response from the list of users, able to create a new user in JSON file
Login in user to Authorization /users/login?Username=[username]&Password=[password] Insert data through PARAMS POST no need of Authorization no JSON format JSON format with a JWT TOKEN created (very long). JSON response from the list of users, able to create a JWT TOKEN
Create movie in the list /movies/ POST JSON format of movie or element to add in the list. { "Genre": { "Name": "Genre type", "Description": "a small description of the genre" }, "Director": { "Name": "name director", "Bio": "Small Bio from director" }, "_id": "60a8cd5037f59be3e29b1c9e", Normally generated when movie is created ramdonly "Title": "movie name", "Description": "description from the movie, or sinopsis", "ImagePath": "link to the image", "ForKids": true, "featured": "if its featured in cinemas or not" }, JSON format added to the list Create new movie in JSON file.
request all the movie list /movies/ need Authorization from the user to call it. TOKEN GET none: just need no call, not format needed JSON format of the complete list of movies JSON response returning the complete list of selected movies, no filter
Return all users /users/ GET need Authorization from the user to call it. TOKEN no format JSON format of selected movie JSON response selecting the movie through the director of the movie
Return user by name /users/:name GET need Authorization from the user to call it. TOKEN no format JSON format of selected movie JSON response selecting the movie through the director of the movie
Request movie selected by name /movies/:name GET need Authorization from the user to call it. TOKEN not format needed to call JSON format of selected movie JSON response returning a selected movies form the list of movies
Return movie selected by director name /movies/:director GET need Authorization from the user to call it. TOKEN none JSON format of selected movie JSON response selecting the movie through the director of the movie
Update movie selected by name /movies/:name PUT need Authorization from the user to call it. TOKEN write new strings of the movie in the JSON format { "Genre": { "Name": "new", "Description": "new" }, "Director": { "Name": "new", "Bio": "new" }, "_id": "60a8cd5037f59be3e29b1c9e", ID will remain the same. "Title": "new", "Description": "new", "ImagePath": "new", "ForKids": true, "featured": "new" }, it will overwrite the previous description and return the new data as JSON format of selected movie JSON response returning a selected movie updated
Update users selected by name /users/:name PUT need Authorization from the user to call it. TOKEN write new strings of the movie in the JSON format { "Username": "new", "Password": "new", "Email": "new", "Birthday": "1990/01/01" (date format), "FavMovies" : [] } it will overwrite the previous description and return the new data as JSON format of selected movie JSON response returning a selected user updated
Add movie to user /users/:name/movies/:name/[ID] -- example (60a8cd5037f59be3e29b1c9e) POST need Authorization from the user to call it. TOKEN no JSON format needed, its Updated through the URL it will add the new id directly on the Favorites movies JSON response returning a user updated
Delete user from the list of users DELETE/ /users/:name/ no JSON format needed Succesful message that was deleted remove user in JSON file and from the list forever and ever.
Delete movie from the movie list DELETE/ /movies/:name/ no JSON format needed Succesful message that was deleted remove movie in JSON file and from the list forever and ever.
